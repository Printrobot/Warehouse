Добавь функциональность «Перемещение коробок»
Общая задача
Добавить новую функцию «Перемещение коробок». Функция позволяет оператору перемещать коробки одного заказа (или отдельные группы коробок внутри заказа) на новое место хранения с возможностью расщепления заказа при необходимости.
В меню оператора добавить пункт «Переместить» 
2. Список заказов для перемещения
При открытии страницы отображается список всех активных заказов, у которых есть хотя бы одна коробка со статусом «на складе».
Для каждого заказа показывать:
Номер заказа
Клиент (если есть)
Дата создания
Общее количество коробок
Краткое описание (можно из первой коробки или теги)
Над списком:
Строка поиска (поиск по номеру заказа, клиенту, тегам).
Кнопка «Сканировать QR полки» – при нажатии открывается камера для сканирования QR‑кода полки. Система ищет заказы, у которых хотя бы одна коробка лежит на этой полке, и отображает их в списке (фильтрация).
3. Выбор заказа и отображение групп коробок
При клике на заказ открывается детальный вид:
Заголовок: «Заказ №…».
Список групп коробок (объединённых по одинаковому типу продукции / описанию / тегам).
Каждая группа содержит:
Тип продукции (можно взять из тегов или описания, например «буклет А5», «брошюра»).
Количество коробок в группе.
Текущее место хранения (если несколько коробок лежат в разных местах – отображать «разные места»).
Чекбокс для выбора группы.
Если в группе больше одной коробки, рядом с чекбоксом появляется поле ввода количества (по умолчанию все коробки группы выбраны). Оператор может уменьшить число, чтобы переместить только часть коробок из группы.
4. Расщепление заказа
Если оператор выбирает не все коробки из группы, система должна:
Создать новую группу для этого заказа.
Для идентификации новой группы добавить к номеру заказа суффикс: -1, -2 и т.д. (например, заказ 12345-1, 12345-2).
В истории заказа отразить расщепление.
Оставшиеся коробки остаются в исходной группе.
После расщепления в списке групп появляется новая запись с суффиксом, и оператор может перемещать её отдельно.
5. Выбор нового места
После выбора групп/коробок оператор нажимает кнопку «Переместить выбранное». Открывается интерфейс выбора места (как в шаге 4 регистрации коробки):
После выбора места оператор подтверждает перемещение.
6. Обновление данных
Система для каждой перемещаемой коробки:
Обновляет 
Добавляет запись в историю коробки (кто, когда, откуда, куда).
Добавляет запись в аудит.
Статус коробки остаётся in_stock (не меняется).
После успешного перемещения система показывает сообщение и возвращает к списку заказов (или к детальному виду заказа, с обновлёнными местами).
7. Ограничения
Перемещать можно только коробки со статусом in_stock.
Если заказ уже полностью выдан (completed), он не отображается в списке.
8. Интерфейс (UX)
Все кнопки крупные, для работы в перчатках.
Для сканирования QR использовать уже готовый компонент QrScanner.
Для камеры и выбора места – переиспользовать логику из (шаг 4).
9. Дополнительные требования
Функция должна быть доступна только для роли оператор.
Все действия фиксируются в журнале аудита.
При расщеплении заказа создавать новую сущность в базе (если заказ изначально был один, то после расщепления мы имеем два заказа с одинаковым базовым номером и суффиксами). В UI показывать их как отдельные записи, но можно объединять в одном списке с группировкой.
10. Пример сценария
Оператор открывает «Переместить».
Сканирует QR полки, видит заказы на этой полке.
Выбирает заказ №12345.
В заказе две группы: «буклет А5» – 5 коробок, «календарь» – 2 коробки.
Оператор выбирает чекбокс «буклет А5» и указывает количество 3 (из 5).
Нажимает «Переместить».
Система расщепляет: исходные 5 коробок «буклет А5» остаются, создаётся новая группа (заказ 12345-1) с 3 коробками.
Оператор сканирует QR новой полки, подтверждает.
3 коробки перемещены, место обновлено.
В деталях заказа 12345-1 отображаются 3 коробки на новой полке.
